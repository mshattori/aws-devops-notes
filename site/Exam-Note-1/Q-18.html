問題18 正解

シナリオ（日本語要約）

  • ある会社では、AWS アカウント内に多数の Amazon EC2 インスタンス を保有し、そこに企業向け Web アプリケーションをホストしている。
  • 会社はクラウドベースのメッセージングプラットフォームとして Slack を使用しており、更新情報やシステムアラートの通知に利用している。
  • 新しい要件として、すべての AWS スケジュール済みメンテナンス通知（AWS が主導する EC2 などへの変更）を、会社の Slack チャンネルに自動で送信する仕組みを構築する必要がある。

質問

  • 上記要件を満たすために、最も簡単（EASIEST）な方法 はどれか。

選択肢（日本語訳）

  • 選択肢1（正解）

    • AWS Personal Health Dashboard と Amazon CloudWatch Events（EventBridge） を組み合わせて、AWS がリソースに対して行うアクティビティをトラッキングする。
    • CloudWatch Events ルールを作成し、そのイベントから AWS Lambda 関数 を起動して、会社の Slack チャンネルへ通知を送信する。

  • 選択肢2

    • Amazon EC2 Events と CloudWatch を組み合わせて、AWS がリソースに対して行うアクティビティを監視する。
    • CloudWatch アラームを作成し、そのアラームから Lambda 関数を起動して Slack チャンネルへ通知する。

  • 選択肢3

    • AWS Config と AWS Trusted Advisor を組み合わせて、AWS が開始した変更をトラッキングする。
    • AWS Config ルールを作成し、違反イベントをトリガーとして Lambda 関数を起動し、Slack チャンネルへ通知する。

  • 選択肢4

    • AWS Support API と AWS CloudTrail を組み合わせて、リソースに対する AWS のアクティビティをモニタリングする。
    • CloudTrail 用の新しいトレイルを作成し、CloudWatch アラームから Lambda 関数を起動して Slack チャンネルへ通知する。

正解と注釈

  • 正解

    • 選択肢1：Personal Health Dashboard + CloudWatch Events（EventBridge）+ Lambda + Slack。

  • この解決策が要件を満たす理由

    • AWS Personal Health Dashboard は、AWS 側が実施するメンテナンスや障害など、アカウント固有の AWS イニシエートイベント を提供するサービスであり、「AWS がスケジュールしたメンテナンス通知」を取得するのに最適。
    • Personal Health Dashboard のイベントは CloudWatch Events / EventBridge に発行できるため、イベントルールでフィルタリングし、対象となる AWS メンテナンスイベントのみを捕捉できる。
    • EventBridge から Lambda 関数 を起動し、Slack Webhook などを使ってチャンネルへメッセージを送信する構成はシンプルかつ標準的であり、「最も簡単（EASIEST）」という条件に合致する。

  • 他の選択肢が不適切な理由（要点）

    • 選択肢2
      • EC2 Events と CloudWatch アラームは主に リソースメトリクスや状態変化 を監視する用途であり、「AWS 主導のスケジュールメンテナンス通知」には特化していない。
    • 選択肢3
      • AWS Config はリソース設定の変更検出が主目的であり、Trusted Advisor はベストプラクティス違反やコスト最適化提案を行うサービスで、メンテナンス通知の取得には適していない。
    • 選択肢4
      • Support API や CloudTrail は、サポートケースや API コールの監査ログ取得が主な用途であり、「AWS 側が予定しているメンテナンス通知」を直接的に取得するのには回りくどい。
      • Personal Health Dashboard + EventBridge を使う場合と比べると、構成も複雑になる。
