🔍 1. 問題点・誤り・補足すべき箇所

  • 対応リソースの網羅性: Backup Supported Services の代表例を一覧化し、リージョン差異に注意する必要がある。
  • 復旧・コピー戦略の不足: クロスアカウント/クロスリージョンコピー、ライフサイクル(Cold Storage/Expire)が未記載。
  • 監査とロックの活用: Audit Manager、Vault Lock の目的と制約を補足し、コンプライアンス対応を明示。
  • 増分バックアップの例外: DynamoDB/Aurora/DocumentDB/Neptune 以外増分である点の理由と注意を強調。

🛠 2. 改訂版（修正・補足反映後の完成文）

  • 対応サービス例: EBS/AMI、RDS & Aurora、DynamoDB、EFS、FSx、Storage Gateway、DocumentDB/Neptune など。対応有無はリージョンで確認。
  • バックアッププラン: スケジュールとライフサイクルを定義し、Warm→Cold→Expire を自動管理。タグベースでリソース自動紐付け。
  • コピー戦略: クロスアカウント/クロスリージョンコピー をサポートし、DR や多層防御を実現。
  • 暗号化: KMS 統合。コピー先では別キーを指定可能。
  • 監査・コンプライアンス: AWS Backup Audit Manager でポリシーチェックを自動化。Vault Lock (WORM) で削除や保持期間短縮を防止。
  • 増分バックアップの例外: DynamoDB/Aurora/DocumentDB/Neptune はサービス固有スナップショット方式。その他は増分で転送・保存を最適化。
