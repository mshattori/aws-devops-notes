  • Personal Health Dashboard と Health API の違い、EventBridge 連携など通知経路が未説明。

🛠 2. 改訂版（修正・補足反映後の完成文）

  • スコープ: Personal Health Dashboard は自アカウント影響のみ。Organizations では委任管理者を指定し Health の組織ビューで全アカウントのイベントを集約。
  • 通知: EventBridge ルールで障害/メンテ/アクション必要イベントを SNS、Incident Manager、OpsItem へ転送し自動対応を開始。
  • フィルタ: サービス名、リージョン、ステータスで絞り込み。履歴は 90 日保存。
  • ユースケース: メンテナンス計画の自動チケット化、重大障害時の運用 runbook 起動、影響サブセットへの段階的対応。
