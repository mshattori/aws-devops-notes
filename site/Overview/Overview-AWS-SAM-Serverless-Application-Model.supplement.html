🔍 1. 問題点・誤り・補足すべき箇所

  • SAM テンプレートと CloudFormation の関係、変換プロセス（Transform）が未説明。
  • sam build/package/deploy/pipeline など CLI ワークフローやローカルテスト(SAM CLI + Local Lambda/API)の補足が必要。
  • デプロイ時の権限（CodeDeploy/CloudFormation ロール）、Stage 切替や自動ロールバックなど実運用ポイントが不足。

🛠 2. 改訂版（修正・補足反映後の完成文）

  • 変換: SAM テンプレートは Transform: AWS::Serverless-2016-10-31 で CloudFormation に展開される。サーバレスリソースを簡潔に定義。
  • CLI ワークフロー: sam build で依存を解決し、sam deploy --guided でスタック作成。sam local start-api/invoke でコンテナ実行によるローカルテスト。
  • デプロイ戦略: CodeDeploy 連携で Lambda の Canary/Linear トラフィックスイッチと自動ロールバックを指定。必要な IAM ロールを CloudFormation Execution Role/CodeDeploy Role として設定。
  • パラメータ/環境: sam deploy --config-env でステージ別設定、S3 アーティファクトバケットを共有。Observability 用に Application Insights/Tracing/Logs 出力を有効化。
