🔍 1. 問題点・誤り・補足すべき箇所

  • Security Hub が集約するファインディングソース（GuardDuty/Inspector/Macie/Config/WAF など）と ASFF 形式が未説明。
  • 標準の有効化（CIS、Foundational Security Best Practices、PCI）とスコープ/抑止期間の設定が不足。
  • マルチアカウント集約（管理/委任管理者、跨リージョン）と自動アクション（Automation/Playbook）への接続を補足すべき。

🛠 2. 改訂版（修正・補足反映後の完成文）

  • 前提: Organizations で管理アカウントまたは委任管理者を設定し、メンバーを自動招待。クロスリージョン集約を有効化可能。
  • 標準: CIS v1.2/1.4、Foundational Security Best Practices、PCI DSS を選択し、必要なサブスクリプションを有効化。
  • ファインディング: GuardDuty/Inspector/Macie/Config/WAF/Firewall Manager/Partner ASFF を統合。抑止期間を設定し、ステータスを管理。
  • 対応: EventBridge ルールや Security Hub 自動化で SSM Automation/Step Functions/ChatOps へ連携し是正。Insights で集約ビューを作成。
