問題11 正解

シナリオ（日本語要約）

  • 政府機関はオンプレミスネットワーク上に、VMware ベースの自動サーバービルドシステムを持っている。
  • このシステムはアプリケーションのサーバーイメージを作成できる仮想化ソフトウェアを利用している。
  • DevOps エンジニアは、オンプレミスのサーバーパイプラインで作成したサーバーイメージを Amazon EC2 上のビルドと挙動にできるだけ近づけてテストできる仕組みを用意するよう依頼されている。
  • これにより、アプリケーションの機能性、非互換性の有無、新しく AWS で使う Amazon Linux 2 OS 上での前提条件を検証したい。

質問

  • 新しい Amazon Linux 2 を前提に、本番で利用する EC2 環境と同等のビルド・挙動をオンプレミスで再現してテストするには、どの解決策を実装すべきか。

選択肢（日本語訳）

  • 選択肢1

    • 最新の AmazonLinux2.iso をダウンロードし、オンプレミスネットワークにインポートする。
    • 仮想基盤を使わず、この ISO を元に新しいオンプレミスサーバーを直接起動し、アプリケーションをデプロイしてテストを行う。

  • 選択肢2

    • AWS Outposts を使ってハイブリッドクラウド環境を構成する。
    • Outposts サーバーに Linux 2 オペレーティングシステムをインストールし、オンプレミスネットワークに接続する。
    • Outposts 環境上の Linux 2 サーバーにアプリケーションをデプロイしてテストを行う。

  • 選択肢3（正解）

    • AWS 上で最新の Linux OS を使用した EC2 インスタンス を起動する。
    • AWS VM Import/Export サービスを使って EC2 イメージをエクスポートし、Amazon S3 バケット内の VMware 用 ISO 形式として保存する。
    • その ISO をオンプレミスサーバーにインポートして新しい仮想マシンとして起動し、アプリケーションをデプロイしてテストを開始する。

  • 選択肢4

    • 任意の Linux ディストリビューション（CentOS / Ubuntu / Fedora など）で新しいオンプレミスサーバーを構築する。
    • これらは技術的に同じであるとみなし、アプリケーションをデプロイしてテストを行う。

正解と注釈

  • 正解

    • 選択肢3：EC2 イメージを VM Import/Export でオンプレミスに持ち込む方法。

  • この解決策が要件を満たす理由

    • 実際に AWS 上の Amazon Linux 2 EC2 インスタンス からイメージを作成し、そのイメージをそのまま VMware へエクスポートするため、OS・パッケージ・設定が本番環境とほぼ完全に一致する。
    • 既存の VMware ベースの自動ビルドパイプラインに統合できるため、オンプレミスと EC2 で同じビルドプロセスと挙動を検証できる。
    • 新しい Amazon Linux 2 上での互換性問題や前提条件を、実際に本番で使用するイメージを用いて確認できる。

  • 他の選択肢が不適切な理由（要点）

    • 選択肢1
      • 物理サーバーに直接インストールする構成であり、VMware ベースの既存パイプラインと整合しない。
      • EC2 の AMI とは別物のインストールとなるため、本番 EC2 環境との完全な一致が保証されない。
    • 選択肢2
      • AWS Outposts の導入はハードウェアコストや運用負荷が大きく、「EC2 と同じ OS イメージでオンプレミス検証を行いたい」という要件に対してオーバースペック。
      • 既存の VMware ベースの仕組みを活かせていない。
    • 選択肢4
      • CentOS / Ubuntu / Fedora は Amazon Linux 2 と「技術的に同じ」ではなく、パッケージ・ライブラリ・設定の違いにより挙動差が生じる可能性が高い。
      • 本番環境（Amazon Linux 2）での問題を十分に検出できない。
