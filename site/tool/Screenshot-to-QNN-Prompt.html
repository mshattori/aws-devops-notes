Screenshot → Q-NN.md Task Specification (for Codex CLI)

This file defines how the Codex CLI assistant should behave when you ask it to convert AWS exam question screenshots into Q-NN.md Markdown files.

When you want to run this task again, you can simply say in a new CLI session:

Run the “Screenshot to Q-NN task” as defined in tool/Screenshot-to-QNN-Prompt.md.

The assistant must then follow the specification below.

Environment assumptions

  • Current working directory: this AWS-DOP Obsidian vault (the notes repository).
  • PNG screenshots of questions are located in ~/Desktop/Screenshot.
  • Each screenshot contains:
  • A header like 問題11 正解 or 問題15 不正解.
  • An English scenario.
  • Four answer options, with the correct one indicated in the UI.

Overall task

  • For every screenshot file under ~/Desktop/Screenshot, read the image and create a Markdown file in the current working directory named Q-NN.md, where:
  • NN is the numeric question ID shown in the header (問題11 → Q-11.md).
  • The content of all generated Markdown files must be written in Japanese, except for AWS service names and technical terms, which can remain in English.

Required structure of each Q-NN.md

Each file must contain at least the following sections.

  1. Title
  2. A level-1 heading with the question number and correctness status.

  3. Example: # 問題11 正解 or # 問題15 不正解.

  4. 問題文と選択肢（日本語要約）

  5. Short Japanese summary of the English scenario.
  6. Translation / rephrasing of each answer option into Japanese.

  7. Use concise, exam-oriented bullet points.

  8. 正解の選択肢と注釈

  9. Clearly identify the correct option.
  10. Add annotations tailored to the problem:
    • Why this option satisfies all explicit requirements.
    • Important AWS services, features, and design patterns involved.
    • Why each other option is inappropriate (1–2 short bullets per wrong option).

  11. Use bold or inline code for key AWS terms and decision keywords.

  12. 不正解と表示されている場合の追加注釈

  13. If the screenshot header indicates the question was answered incorrectly (不正解), add a dedicated section explaining the wrong choice:
    • Explicitly state which option was selected but is incorrect.
    • Explain clearly why it does not meet the requirements.
    • Provide 1–3 bullet “学習ポイント” to avoid the same mistake.

Special handling for Question 15

For 問題15 (file Q-15.md):

  • Always include a subsection such as ## 自分が選んだ不正解の回答.
  • In that subsection:
  • Clearly state which option was chosen in the exam.
  • Explain in Japanese why that option does not satisfy the scenario requirements.
  • Add a short list of “学習ポイント” focusing on:
    • Deployment strategy choice (in-place vs blue/green).
    • CodeDeploy deployment configuration (e.g., AllAtOnce, custom settings).
    • Lifecycle hooks (BeforeAllowTraffic, BlockTraffic, etc.).

Formatting rules

  • Follow the existing heading style used in other Q-NN.md files in this repo.
  • Prefer bullet lists to long paragraphs for explanations.
  • Mix Japanese explanations with original English AWS terminology naturally.
  • Ensure each file is self-contained and useful for quick pre-exam review.
